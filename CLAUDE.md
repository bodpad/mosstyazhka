# МосСтяжка — сайт-визитка

Статичный сайт компании по полусухой стяжке пола в Москве и Подмосковье.

## Контекст бизнеса

- **Название:** МосСтяжка
- **Услуга:** механизированная полусухая стяжка пола с фиброволокном
- **География:** Москва и Подмосковье, выезд до 150 км от МКАД
- **Замер:** бесплатный, на любой адрес в зоне выезда
- **Опыт:** с 2017 года
- **Гарантия:** 3 года, закреплена в договоре
- **Юридическая форма:** ИП на УСН 6% (планируется регистрация)
- **Домен:** мосстяжка.рф

## Целевая аудитория

1. **Частные клиенты** — собственники квартир в новостройках, владельцы загородных домов
2. **Дизайнеры и прорабы** — ищут надёжного подрядчика для своих объектов
3. **Юрлица** — застройщики и коммерческие помещения, которым нужны закрывающие документы

Большинство клиентов — мужчины и женщины 30–55 лет, делают ремонт. Решение часто принимается семьёй, поэтому сайт должен быть понятен и человеку без строительного опыта.

## Технический стек

- HTML5 + **Tailwind CSS** (компилируется через Tailwind CLI, output.css коммитится в репо)
- Минимум JavaScript — только если без него никак (форма, мобильное меню, FAQ-аккордеон)
- Адаптивная вёрстка (mobile-first): телефон → планшет → десктоп
- Размещение: Cloudflare Pages
- Без бэкенда. Формы отправляются через сторонние сервисы (Formspree, Getform) или напрямую в WhatsApp/Telegram через ссылку.

### Подключение Tailwind

Tailwind собирается локально через CLI. Конфиг — `tailwind.config.js`, исходник — `input.css`, результат — `output.css` (минифицированный, ~13 КБ).

```bash
npm install       # ставит tailwindcss из package.json
npm run build     # собирает output.css один раз (для коммита)
npm run watch     # пересобирает на лету при изменениях
```

`output.css` коммитится в репозиторий — Cloudflare Pages раздаёт его как статику без build-шага. После изменения классов в `index.html` нужно запустить `npm run build` перед коммитом.

В `index.html` подключается одной строкой:

```html
<link rel="stylesheet" href="output.css">
```

Кастомизация цветов и шрифтов — в `tailwind.config.js` (секция `theme.extend`).

На старте — однастраничный лендинг (index.html). Многостраничник добавляем только если появится реальная необходимость.

## Дизайн-система

### Цветовая палитра (через Tailwind config)

Расширяем стандартную палитру в `tailwind.config.js`:

```js
theme: {
  extend: {
    colors: {
      dark: '#1A1A1A',      // основной текст, кнопки, шапка
      accent: '#F5C518',    // акценты, CTA, выделения (жёлтый)
      cream: '#FAFAF8',     // фон страницы
      muted: '#666666',     // второстепенный текст
    },
    fontFamily: {
      sans: ['system-ui', '-apple-system', 'Segoe UI', 'Roboto', 'sans-serif'],
    },
  },
}
```

Использование в HTML: `bg-dark`, `text-accent`, `border-muted` и т.д. После добавления новых классов в `index.html` запускай `npm run build`.

Цветовая схема — чёрно-жёлтая, как в логотипе. Это классика строительного брендинга, вызывает ассоциации с надёжностью и предупреждающими знаками (DeWalt, Caterpillar).

### Типографика

- Базовый шрифт — системный, через `font-sans` (настроен в конфиге выше)
- Заголовки: `font-extrabold` или `font-black` (как в логотипе)
- Текст: `font-normal`
- Размеры: `text-base` (16px) на мобильном, `md:text-lg` (18px) на десктопе
- Межстрочный интервал: `leading-relaxed` (1.625)
- Заголовки H1: `text-4xl md:text-6xl font-black`
- Заголовки H2: `text-3xl md:text-4xl font-extrabold`
- Заголовки H3: `text-xl md:text-2xl font-bold`

### Компоненты (паттерны Tailwind)

- **Кнопка CTA основная:** `bg-dark text-accent px-6 py-3 rounded-lg font-bold hover:bg-dark/90`
- **Кнопка CTA вторичная:** `bg-accent text-dark px-6 py-3 rounded-lg font-bold hover:bg-accent/90`
- **Кнопка-обводка:** `border-2 border-dark text-dark px-6 py-3 rounded-lg font-bold hover:bg-dark hover:text-cream`
- **Карточка:** `bg-white rounded-xl p-6 shadow-sm border border-gray-200`
- **Контейнер:** `max-w-6xl mx-auto px-4 md:px-8`
- **Секция:** `py-12 md:py-20`

### Правила оформления

- **Без градиентов, без размытия, без декоративных эффектов** — строгий, утилитарный вид
- **Тени минимальные** — `shadow-sm` или вообще без них, чтобы сайт не выглядел «пузырчатым»
- **Скругления среднего размера** — `rounded-lg` для кнопок, `rounded-xl` для карточек
- **Не использовать произвольные значения** через `[...]`, если есть подходящий стандартный класс
